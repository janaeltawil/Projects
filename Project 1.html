Abstract

In this project, I performed a Random Forest classification to determine whether certain musical features influenced a track’s popularity on the TikTok app. I selected eight predictor features that I deemed appropriate for the analysis's sake. I then categorized certain features for the purpose of classification. Ultimately, I found the Decision Tree provided a moderately higher accuracy level than the Naive Bayes classification. I also analyzed the variable importance plot, according to the random forest classification.

 

Introduction

            The debate surrounding social media algorithms and the homogenization of music taste has long been discussed from a musical, cultural, and behavioral standpoint. Such discussions have focused on the role of algorithms in bolstering certain songs to garner mainstream appeal, which translates into increased viewership and advertisement revenue (Caramanica, 2024).With this premise in mind, most of these discussions conclude that the algorithms of YouTube, TikTok, and Spotify tend to favor tracks that appeal to the broad majority of users, as opposed to more niche or experimental genres (Wang, 2024). As such, many have argued that algorithms have an echo-chambering effect, simultaneously picking tracks that are likely to attract more users, and implicitly informing users’ collective music preferences uniformly (Wang, 2024). Given this conventional wisdom, I decided to focus on the TikTok app due to its contemporary relevance. I chose features that described tracks from a strictly musical perspective (i.e. tempo, key, mode, etc.) to determine whether such features had a bearing on a track’s popularity in TikTok’s chart. The dataset used was sourced from Kaggle, and it includes the top-charting tracks from the TikTok app and the song’s musical features, courtesy of Spotify’s API (Peleg, 2021).

 

 

 

 

 

 

Materials and Methods

            The dataset has 23 features and 6746 observations, each representing a popular track on the TikTok app. It is much larger than my previously used dataset which contained 267 entries. Hopefully, the larger dataset will yield an advantage since random forests are particularly effective at dealing with large, multidimensional feature spaces by scaling them. I was mainly interested in the same features I chose for my first project as predictor variables:

  1. Tempo: This refers to the track’s speed or a beat’s duration, measured in beats per minute (BPM).
  2. Mode: This is the track’s modality which is either “major” or “minor”, coded as a binary variable with ‘1’ for “major” and ‘0’ for “minor”.
  3. Key: This is an estimate of the track’s overall key, according to Pitch Class notation.
  4. Speechiness: This measures the occurrence of spoken words in a track, measured from 0.0 to 1.0 with 1.0 referring to more speech in the recording. 
  5. Acousticness: This is also a measure from 0.0 to 1.0 that is a confidence estimate of a track being acoustic.
  6. Instrumentalness: This predicts a track containing vocals or not, but it distinguishes certain vocals such as “ooh” and “aah” as instrumental.
  7. Liveness: This is a measure that detects a live audience’s presence in the track recording, as well as the probability that the track was performed live.
  8. Valence: This describes how emotionally positive a track is compared to a low valence which indicates more sadness or anger (Shetty, 2018).

The main difference in my cleaning process here is that I will not be categorizing all variables because that is not needed for random forests. Instead, I categorized the mode and key features (since they are coded as integers), as well as my response variable ‘popularity’ for the sake of classification. The rest of the features remain numeric.

Results

  1. K-fold Cross Validation Analysis

After creating a data frame with my newly categorized features, I split the data using 75% for the training data and 25% for the testing data. This left me with a training set of 5059 observations and a testing set of 1687 observations. I then performed K-fold cross-validation on my training set, splitting it into 10 folds. The tuning parameter (mtry) was tested for values of (2:5), and the best value for the highest accuracy was 5.

 

2.              Random Forest Classification Analysis

After performing K-fold cross-validation, I performed the Random Forest Analysis on my training set, using 1000 trees, and 5 variables for each split. The out-of-bag error rate (OOB) was 25.91%, indicating that the accuracy was around 74.09%. The class error rates were highest for the “Unpopular” class (0.628) and “Moderately Popular” class (0.561) compared to the lower rates of the “Very Popular” class (0.16) and “Popular” class (0.086). Despite the larger error rates for the first two classes, I was still satisfied with the model’s performance since my main goal was to determine the higher levels of popularity, based on the features. Surprisingly, the model seemed to overfit the “Unpopular” observations to the “Popular” class (294 instances) while the correctly identified instances of “Unpopular” were lower (236 instances). More expectedly, the model overfits the “Moderately Popular” observations between the correct class (401 instances) and the “Popular” class (439) which makes more sense since they are closer in level.

 

With the training set complete, I finally fit the model on my testing set of 1687 observations. Overall, the accuracy was 74.39%, marginally higher than the training set’s accuracy. This suggests the model’s relative stability. The sensitivity rate was highest for the “Popular” class (91.4%) and “Very Popular” class (84.4%) whilst being lower for “Moderately Popular” (46.62%) and “Unpopular” (39.46%) classes. This indicates the model’s stronger ability to predict True positives in those first two classes. The specificity was also high across all classes except for the “Popular” one which had a rate of 67.85% compared to other classes’ rates ranging from 95-99%. This indicates the model’s weakness in determining true negatives for the “Popular” class which also makes sense since the training set tended to overfit incorrect observations to that class. The accuracy was highest for the “very Popular” class (90.53%), followed by the “Popular” class (79.63%) while the “Moderately Popular” (71.53%) and “Unpopular” (68.91%) classes were lower.

Finally, I used the Gini index to evaluate the importance of each feature and their reduction of impurity at each split. Acousticness seems to have been the most influential predictor with a Mean Decrease Gini value of 564.04, followed by Speechiness (559.00), tempo (555.29), and valence (542.79). Mode seems to have been the least important predictor with a value of 54.72, suggesting its lack of contribution to the splits.

 

Discussion

Overall, it seems that the model performed moderately well when considering all classes, giving it an accuracy of approximately 74%. However, when focusing on the “Popular” and “Very Popular” classes, its strength in performance is more pronounced. I found this to be a strength for my chosen topic since the goal was to analyze what boosts a song’s popularity. The model’s performance was much better than my previous models using Decision Tree (accuracy rate of 43.04%) and Naïve Bayes model (accuracy rate of 41.77%.). However, it is worth mentioning that these two models used a significantly smaller sample size. The “Moderately Popular” and “Unpopular” classes had high misclassification rates which was a weakness of the model. This was also the case with the Naïve Bayes and Decision Tree models as both struggled the most with those same two classes. Despite performing K-fold cross-validation, perhaps I should have addressed the class imbalance of the sample by using an under-sampling technique or even including class weights. I did initially set class weights to be used in the ‘classwt’ input of the random forest function but my accuracy was somewhat lower (73.74%) overall, and the performances across classes were either worse or unchanged so I decided to not use it.  Further, perhaps it would be better to focus on the more important features such as “acousticness” and “speechiness” while dropping less influential ones like “mode” or “key”. Interestingly, the decision tree performed for my first project seemed to rely on the “key” feature across numerous splits.
